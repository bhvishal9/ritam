## Conclusion

## Threshold sweep summary (this dataset, gemini-embedding-001)

| 0.80 | 0.222 | 0.000 | 0.933 | 0.206 |
| 0.85 | 0.000 | 0.000 | 1.000 | 0.000 |

There is no single threshold on `gemini-embedding-001` that simultaneously gives high factual recall *and* meaningful abstention on this corpus. The recall cliff between 0.70 and 0.80 is the diagnostic finding — it says the in-domain and out-of-domain similarity distributions overlap heavily under this embedding model, so a single scalar threshold cannot separate them. Real fixes (in priority order, deferred until after cost-control work):

1. Switch embedding model and re-sweep — different models produce different score distributions; a model where in-domain queries cluster nearer 0.85 changes the picture.
2. Add per-doc deduplication / max-per-doc cap to recover multi-hop recall independently of threshold.
3. Add a reranker stage so the threshold operates on reranked scores rather than raw embedding similarity. This is also the cheapest path to better abstention without sacrificing recall.
4. Hybrid search (BM25 + dense) — the lexical channel will catch named-entity queries like `f_hard_treaty_long_hum` that pure dense retrieval misses.
